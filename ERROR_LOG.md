[2026-03-28 19:00] - Agent: QA Agent
Error: `app/api/universities/[slug]/route.ts` uses deprecated `params` destructuring pattern.
Context: The handler signature is `{ params }: { params: { slug: string } }` (line 6) and accesses `params.slug` synchronously. In Next.js App Router (newer versions), `params` is a Promise and must be awaited. Accessing it synchronously will throw or silently return undefined in the affected version, causing all detail-page requests to fail.
Solution: Update the handler to `{ params }: { params: Promise<{ slug: string }> }` and `const { slug } = await params;` before use.
Prevention: Read `node_modules/next/dist/docs/` for the current dynamic-route params API before writing any route handler. The AGENTS.md instruction explicitly flags this.

Prevention: All catch blocks in API routes must log the error before returning the error response. Establish a shared error-handler utility.

[2026-03-28 19:00] - Agent: QA Agent
Error: `lib/queries/university.ts` — `tuition_max` filter only applied when `degree`, `field`, or `scholarship` are also set.

[2026-03-28 19:00] - Agent: QA Agent

Solution: After mapping `universities` to `mapped`, add a post-query sort: `if (sort === 'tuition_asc') mapped.sort((a, b) => (a.tuition_from_myr ?? Infinity) - (b.tuition_from_myr ?? [2026-03-28 19:00] - Agent: QA Agent
Error: `lib/queries/university.ts` — `getUniversityBySlug` converts potentially-null `coordinates_lat` / `coordinates_lng` to `Number(null)` = 0 without a null guard.
Context: Lines 192-193: `coordinates_lat` and `coordinates_lng` are `Decimal` (non-optional) in the Prisma schema, but the `Number()` cast on a `null`-like Prisma Decimal would produce `0` rather than `null`. If the DB row somehow has a zero coordinate it is indistinguishable from a missing value. The TypeScript interface (`UniversityDetail`) types them as `number` (not `number | null`), which is consistent with the schema, but the cast is fragile if the schema changes.
Solution: This is low-severity given the schema marks both fields non-nullable, but adding an explicit assertion (`if (!uni.coordinates_lat) throw new Error(...)`) would surface data issues earlier.
Prevention: Prefer explicit Prisma `Decimal` → `number` conversion helper with null safety.

[2026-03-28 19:00] - Agent: QA Agent
Error: `lib/queries/search.ts` — `getSearchSuggestions` returns an object `{ universities, programs }` but the route handler wraps it as `{ success: true, data: suggestions }`, making the response shape `{ success: true, data: { universities: [...], programs: [...] } }`. The test BE-020 (empty array for q < 2 chars) expects `data: []` but the actual response for empty input is `{ success: true, data: [] }` (the function returns `[]` directly). Prevention: Define and enforce a consistent response schema for each endpoint; add a TypeScript return type to `getSearchSuggestions`.
